github-pages去除custom-domain配置后，浏览器依旧重定向的解决方案

 2022-02-12  1 min  GitHub

前言：之前做了 GitHub 的 github-pages 的链接跳转，后来解除跳转之后，浏览器一直还是在重定向到个人域名，后发现是浏览器缓存了重定向配置

解决方案，清除浏览器缓存

  • 1.浏览器设置里面清除所有缓存
  • 2.清除浏览器的 dns 缓存等
    • Url 输入 chrome://net-internals/#dns 点击Clear host cache
    • Url 输入 chrome://net-internals/#sockets 点击Flush socket pools
